# 阅读回访与知识调用系统

建立日期：2026-06-02

这个系统的目标不是保存更多摘录，而是把微信读书里的阅读痕迹变成可以反复调用的知识资产。

## 核心循环

1. 捕获：从微信读书获取最近阅读、笔记本概览、单本书划线和想法。
2. 回访：按 0 天、3 天、14 天、45 天节奏重新面对一本书或一组笔记。
3. 提炼：把有用内容整理成观点卡、方法卡、案例卡、问题卡、引用卡。
4. 调用：围绕写作、决策、工作、自我反思、对话准备来召回卡片。

## 目录

- `00-inbox/`：临时收集区，放刚导出的划线、想法、未处理材料。
- `01-reviews/`：阅读回访记录，每本书一份或一次回访一份。
- `02-cards/`：可调用知识卡片。
- `03-topics/`：主题索引，把多本书、多张卡片串起来。
- `04-workflows/`：固定工作流，比如周回访、写作调用、决策调用。
- `05-snapshots/`：微信读书数据快照。
- `templates/`：复用模板。

## 第一版使用方式

每周做一次轻量回访：

1. 打开 `04-workflows/weekly-review.md`。
2. 从 `05-snapshots/2026-06-02-weread-overview.md` 里选 3 本书。
3. 用 `templates/review-card.md` 为每本书生成一张回访卡。
4. 每本书只提炼 1-3 张 `02-cards/` 知识卡。
5. 把卡片挂到 `03-topics/topic-index.md` 的主题下面。

## 发布为网站

这个目录是纯静态网站，可以直接部署到 GitHub Pages。

推荐方式：

1. 在 GitHub 新建一个仓库，比如 `reading-system`。
2. 把本目录内容推送到仓库。
3. 进入仓库 `Settings` → `Pages`。
4. `Build and deployment` 选择 `Deploy from a branch`。
5. 分支选择 `main`，目录选择 `/ (root)`，保存。
6. 等待 Pages 构建完成后，用 GitHub 提供的网址访问。

如果你把上一级项目目录作为仓库根目录发布，根目录的 `index.html` 会自动跳转到 `reading-system/`。

## 调用口令

以后你可以直接在对话里说：

- “从我的读书笔记里找和长期主义有关的内容”
- “帮我回访最近读过的 3 本书”
- “把《反脆弱》的笔记整理成方法卡”
- “我在写一篇关于教育的文章，调用相关阅读材料”
- “本周给我一个阅读回访清单”
